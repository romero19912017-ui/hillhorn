# Hillhorn — рекомендации по развитию до полноценной IDE уровня Cursor+

**Цель:** среда разработки уровня Cursor и выше с NWF-памятью, JEPA и DeepSeek.

---

## 1. Управление памятью (NWF)

| Фича | Описание | Зачем |
|------|----------|-------|
| **Очистка памяти** | Pruning старых зарядов по возрасту, лимит размера поля (например, 10k зарядов), сжатие sigma для "забывания" | Рост data/deepseek_memory без ограничений — утечка ресурсов, замедление поиска |
| **Экспорт/импорт памяти** | Сохранение поля в файл, загрузка для другого проекта | Бэкапы, перенос контекста между машинами |
| **Раздельная память по проектам** | Каждый workspace — своё NWF-поле | Контекст проекта A не смешивается с B |
| **Приоритизация зарядов** | Учёт времени, успешности, alpha при поиске | Более релевантные результаты |

---

## 2. Индексирование кодовой базы

| Фича | Описание | Зачем |
|------|----------|-------|
| **Индексация новых проектов** | При открытии workspace — сканирование *.py, *.ts, *.js и др., эмбеддинги функций/классов/модулей | "Explain this project", семантический поиск по коду |
| **Автоиндексация при сохранении** | File watcher: на onSave — обновить эмбеддинг изменённого файла | Актуальный контекст без ручного sync |
| **Codebase graph** | Граф зависимостей (импорты, вызовы) | Навигация "кто использует эту функцию" |
| **Исключения из индекса** | .gitignore-подобный список (node_modules, venv, __pycache__) | Экономия ресурсов, меньше шума |

---

## 3. UI и интеграция в редактор

| Фича | Описание | Зачем |
|------|----------|-------|
| **Расширение VS Code (Hillhorn Chat)** | Панель чата в сайдбаре, WebSocket к Gateway | Чат в редакторе, как в Cursor |
| **Inline-подсказки** | Copilot-подобные предложения при наборе | Быстрее писать код |
| **Стриминг в UI** | Постепенное отображение ответа (chunk by chunk) | Ощущение скорости, можно прервать |
| **Контекст из редактора** | Передача в запрос: открытые файлы, курсор, выделение | Агент знает, над чем работаешь |
| **Apply edits** | Кнопка "применить" — вставить сгенерированный код в файл | Меньше copy-paste |

## 5. Агенты и инструменты (Function Calling)

| Фича | Описание | Зачем |
|------|----------|-------|
| **Function calling** | tools: search_memory, read_file, write_file, execute_command, call_agent | Автономность: Planner сам ищет в памяти, вызывает Coder |
| **Безопасность execute_command** | Whitelist команд, sandbox, подтверждение опасных операций | Защита от вредоносных действий |
| **ArchitectAgent, DocumenterAgent** | Классы в agents.py, уже замаплены в Gateway | Полный набор агентов |

---

## 6. Интеллект и надёжность

| Фича | Описание | Зачем |
|------|----------|-------|
| **Семантические эмбеддинги** | Один энкодер (sentence-transformers) в Gateway, Adapter, JEPA | Качество поиска вместо хэша |
| **Автовыбор агента** | select_agent_from_memory перед каждым запросом | Маршрутизация по смыслу |
| **Fallback-модели** | При ошибке reasoner — retry на chat | Меньше падений |
| **Кэш частых запросов** | Повторный запрос — ответ из кэша | Экономия API, скорость |
| **Очередь запросов** | При rate limit — поставить в очередь | Стабильность под нагрузкой |
