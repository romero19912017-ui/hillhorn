# Hillhorn - план проверок и анализ эффективности

## Цель

Проверить работу Hillhorn на реальных задачах, замерить эффективность, сформировать выводы.

## 1. Сценарии проверок

### 1.1 Новая задача (холодный старт)
- **Что:** Агент получает задачу в проекте без предзаполненной памяти
- **Шаги:** hillhorn_get_context → hillhorn_search → работа → hillhorn_add_turn
- **Ожидание:** Память пуста, после add_turn — накопление
- **Метрики:** Время, количество вызовов, релевантность результата search

### 1.2 Повторная задача (теплый контекст)
- **Что:** Вторая задача после накопления памяти
- **Шаги:** hillhorn_get_context → hillhorn_search (по теме прошлой задачи) → работа
- **Ожидание:** search находит прошлые решения
- **Метрики:** Релевантность search, экономия итераций

### 1.3 Сложная задача (planner)
- **Что:** Многошаговая задача
- **Шаги:** hillhorn_get_context → hillhorn_consult_with_memory(planner) → план → реализация
- **Ожидание:** Planner дает структурированный план
- **Метрики:** Качество плана, время consult, токены

### 1.4 Ревью кода (reviewer)
- **Что:** Проверка написанного кода
- **Шаги:** hillhorn_consult_agent(reviewer, code_to_review=...) → правки
- **Ожидание:** Конкретные замечания
- **Метрики:** Полезность замечаний, время ответа

### 1.5 Индексация и поиск
- **Что:** Индексируем файл → ищем по теме
- **Шаги:** hillhorn_index_file → hillhorn_search
- **Ожидание:** search находит индексированное
- **Метрики:** Recall поиска

## 2. Структура папки hillhorn_checks

```
hillhorn_checks/
  PLAN.md              - этот план
  REPORT_TEMPLATE.md   - шаблон отчета
  tasks/
    task_01_cold_start/    - новая задача
    task_02_warm_context/  - с памятью
    task_03_planner/       - сложная задача
    task_04_reviewer/      - ревью кода
    task_05_index_search/  - индексация + поиск
  run_all.ps1          - запуск всех проверок
  analyze.py           - скрипт анализа
```

## 3. Метрики эффективности

| Метрика | Как замерить |
|---------|--------------|
| Время до первого результата | От старта до полезного ответа |
| Количество итераций | Вызовов MCP, read_file и т.д. |
| Релевантность search | 0-3: нет/частично/да/отлично |
| Полезность planner | 0-3 по структурированности |
| Полезность reviewer | 0-3 по конкретности |
| API cost (токены) | Логи Gateway / консультаций |
