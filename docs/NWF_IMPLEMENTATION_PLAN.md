# План внедрения NWF по результатам исследований (HABR)

На основе статьи «Нейровесовые Поля: метод хранения данных с калиброванной неопределённостью».

## Что взять в первую очередь

### 1. Симметричное расстояние Махаланобиса (приоритет 1)

**Проблема:** Hillhorn использует L2. В статье: L2 завышает точность, асимметричный Махаланобис — занижает. Правильно — симметричный.

**Формула:**
```
d²_M = (z_q - z_i)ᵀ (Σ_q + Σ_i)⁻¹ (z_q - z_i)
```

**Действие:** Использовать `field.search(query_charge, k)` или `mahalanobis_symmetric` из nwf-core. Для запроса нужен Charge(z_q, sigma_q), а не только z.

**Файлы:** `tools.py` (search_memory), возможно вызов `field.search()` вместо ручного L2.

---

### 2. Ковариация запроса (приоритет 2)

Сейчас при поиске мы передаём только эмбеддинг z. Для Махаланобиса нужны (z_q, Σ_q). Варианты:

- **A:** Задавать Σ_q = diag(σ²) с маленьким σ (напр. 0.2) — «уверенный» запрос
- **B:** Обучать/оценивать Σ для запроса (сложнее)
- **C:** В nwf-core `field.search(charge)` — передавать Charge, а не вектор

---

### 3. Product Quantization (PQ) — опционально

**Из статьи:** Сжатие 49× (512 → 16 байт) при потере ~1% точности.

**Действие:** Использовать `nwf-core` с FAISS PQ для больших хранилищ. Для Hillhorn (десятки тыс. зарядов) можно отложить.

---

### 4. Калибровка уверенности (приоритет 3)

**Метрика:** `agreement_ratio` — доля k ближайших соседей с тем же классом. ECE 0.14→0.03 после Platt scaling.

**Действие:** Добавить в `hillhorn_consult_with_memory` оценку уверенности по agreement соседей по тегам (kind, project).

---

### 5. Взвешивание по потенциалу

**Формула потенциала:**
```
φ_i(r) = exp(-0.5 · (r - z_i)ᵀ Σ_i⁻¹ (r - z_i))
```

**Действие:** Ранжировать результаты не только по расстоянию, но и по φ_i(z_q) — ближе к «центру» заряда = выше вес.

---

### 6. HNSW для масштабирования

**Из статьи:** Ускорение поиска 14–37×. В nwf-core HNSW в планах.

**Действие:** Пока использовать `field.search()`. При росте до сотен тысяч зарядов — рассмотреть FAISSIndex/HNSW из nwf-core.

---

### 7. Обновление весов (alpha)

**Формула поля:** Φ(r) = Σ α_i · φ_i(r). Alpha — вес заряда.

**Действие:** При успешном использовании заряда (найден и помог) увеличивать alpha; при провале — уменьшать. Нужен механизм update в Field или пересохранение заряда с новым alpha.

---

## Что НЕ внедрять сейчас

| Компонент | Причина |
|-----------|---------|
| Kalman-кодирование | В 4–5 порядков медленнее VAE; для Hillhorn хватает hash/sentence-transformers |
| OOD-детекция | Полезно, но не критично для MCP-памяти |
| Семантическая интерполяция | В основном для визуализации |

## Порядок внедрения

1. ~~**Симметричный Махаланобис**~~ — ✅ Реализовано: `field.search(query_charge, k, metric="symmetric")` в tools.search_memory и nwf_memory_adapter.search_similar
2. ~~**Ковариация запроса**~~ — ✅ Реализовано: query_charge = Charge(z=embed(query), sigma=0.2)
3. ~~**Взвешивание по потенциалу**~~ — реализовано: potential = exp(-0.5*d^2) в результатах search_memory
4. ~~**Обновление alpha**~~ — утилита `nwf_memory_utils.boost_charges_alpha(indices, delta)` для вызова при успешном использовании
5. ~~**Калибровка**~~ — реализовано: agreement_ratio в search_memory, confidence в hillhorn_consult_with_memory
6. **PQ/HNSW** — при росте объёма данных
